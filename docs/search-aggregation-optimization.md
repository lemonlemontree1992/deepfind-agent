# 搜索结果综合性优化文档

## 📋 优化概述

### 问题诊断
**原问题**：同一调研主题多次提问，结果不一致
- ❌ 搜索结果随机性强，每次返回不同来源
- ❌ 没有综合多方信息
- ❌ 没有考虑来源权威性
- ❌ 没有统计提及次数

### 解决方案
**综合多方信息，加权排序输出**

---

## 🔧 核心优化

### 1. 搜索结果聚合器 (`tools/result_aggregator.py`)

#### 功能特性

**① 来源于权威性评分（0-10分）**
```
政府网站 (.gov.cn)           → 10分
知名媒体（人民网、新华网）    → 9分
专业平台（马蜂窝、携程）      → 8.5分
知名社区（知乎、豆瓣）        → 7.5分
个人博客                      → 5分
其他                          → 6分
```

**② 关键词匹配评分**
- 标题完全匹配：6分
- 标题部分匹配：按比例评分
- 内容相关性：2分
- 来源权威性：2分

**③ 提及次数加成**
- 同一URL重复出现：每次+0.5分
- 最高加成：2分

**④ 综合评分公式**
```
总分 = 相关性评分 + 权威性评分/10 + 提及次数加成
```

---

### 2. 排序逻辑

#### 原流程
```
搜索 → 去重（URL） → 返回结果
         ↑
      简单去重，无排序
```

#### 新流程
```
搜索 → 去重(URL) → 聚合排序 → 过滤低分 → 返回结果
                      ↓
               综合评分排序
                      ↓
               记录Top 3结果
```

---

### 3. 共识信息提取

#### 功能
从多个来源中提取最常被提及的信息：

```python
consensus = {
    'top_attractions': [...],  # 最常提及的景点（前3）
    'top_restaurants': [...],  # 最常提及的餐厅（前3）
    'top_hotels': [...],       # 最常提及的酒店（前3）
    'common_tips': [...],      # 常见建议（前3）
}
```

#### 用途
在分析阶段优先关注这些共识信息，它们更可能准确可靠。
